## How to work in this repo

1. **Read the specs first:** active changes in `.forge/specs/active/<change-id>/`; current
   verified truth in `.forge/product/current/capabilities/`. Do not edit `product/current/` —
   only `/forge:archive` writes there.
2. **Conventions are rules:** read `.forge/rules/` for the categories your change touches
   (conventions always; architecture for backend; domain for money/audit; testing before tests).
3. **Boundaries:** identifiers in English, docs in Português Brasileiro; money as integer cents;
   no technology prefixes in names; no AI co-authorship in commits (Conventional Commits).
4. **Worktrees:** work in `.forge/worktrees/<change-id>/` (enforced by hook); branch
   `feature/<change-id>`; PR to `develop`.
5. **Validate before done:** run the commands above plus `.forge/scripts/` gates; human gates are
   explicit options recorded in `approvals.yaml`.
